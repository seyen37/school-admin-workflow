# 基礎課程：教師的 NotebookLM 與 Gemini 工作流

> 適用對象：一般教師、第一次接觸 AI 工具的承辦人
> 預計學習時間：**45 分鐘**
> 前置需求：**會用 Google Drive、Gmail 即可**

---

## 為什麼選這兩個工具

學校現場有許多既有的 AI 工具（ChatGPT、Copilot、文心一言…），這份教材聚焦在 **NotebookLM 與 Gemini** 兩個 Google 自家工具，原因有三：

1. **跟 Google Workspace 整合**：你的公文、會議紀錄、學生作業多數已經在 Google Drive 裡。NotebookLM 可以直接讀 Drive 的檔案，不必反覆上傳。
2. **資料留在自己帳號內**：Google 對 Workspace for Education 帳號明示「不會用使用者資料訓練模型」，比第三方工具的隱私顧慮少。
3. **學校多數已有授權**：Workspace for Education Plus 已包含 Gemini，不需要額外付費（請跟學校 IT 確認）。

> ⚠️ **不是反對其他工具**：ChatGPT、Claude 都有它們的強項，但這份教材的目標是「**用學校既有的工具立刻見效**」，所以聚焦在這兩個。

---

## 兩個工具的分工

**簡單版本一句話：**

| 工具 | 它擅長什麼 | 它不擅長什麼 |
|---|---|---|
| **NotebookLM** | 把多份文件變成可以詢問的「**知識庫**」 | 從零生成文字 |
| **Gemini** | 從零生成文字、回答開放問題 | 看你私人文件 |

更白話：

- **NotebookLM 像「你的個人圖書館員」**：你把資料丟給它，它根據資料回答；不會編造資料外的內容。
- **Gemini 像「你的寫作助理」**：你給指令，它寫東西；但它不知道你私人文件的內容。

**正確的使用順序**：
1. 先用 **NotebookLM** 整理脈絡（讀你的公文、附件、會議紀錄）
2. 用 **NotebookLM 摘要 / 對話**形成草稿要點
3. 把要點丟給 **Gemini**，要它寫成完整文章

## NotebookLM 5 分鐘入門

### 1. 開啟 NotebookLM

到 [notebooklm.google.com](https://notebooklm.google.com) → 用學校 Workspace 帳號登入。

### 2. 建立新 Notebook

點「**+ Create new**」，給筆記本起個名字（例：**「2026 春季 AI 融入研習」**）。

### 3. 加入來源資料

NotebookLM 一個 Notebook **最多 50 個來源**，每個來源最多 500k 字。可以加：

- ✅ Google Docs / Sheets / Slides（直接從 Drive 選）
- ✅ PDF 檔案（上傳）
- ✅ 純文字檔
- ✅ 網頁 URL
- ✅ YouTube 影片連結（會抓字幕）
- ✅ 直接貼文字

### 4. 開始提問

加完來源後，可以在輸入框問問題：

```
請整理這份公文的核心要求，列成 5 條 checklist
```

```
這次研習的成果報告需要交哪些附件？依公文整理
```

NotebookLM 的回答**會引用來源段落**，點藍色數字可以跳到原文位置——這是它跟 ChatGPT/Gemini 最大的差別。

### 5. 用「Notebook Guide」快速摘要

點右側「Notebook Guide」可以：

- ✅ **自動生成摘要**（適合新接案時快速理解全貌）
- ✅ **建議的問題**（如果你不知道該問什麼）
- ✅ **學習指南**（給教學情境用）
- ✅ **時間軸**（找跨日期的脈絡）
- ✅ **常見問答 FAQ**

## Gemini 5 分鐘入門

### 1. 開啟 Gemini

到 [gemini.google.com](https://gemini.google.com) → 用學校 Workspace 帳號登入。

### 2. 寫一個有「角色 + 任務 + 約束」的提示

**新手最常犯的錯**：「幫我寫個成果報告」——這太籠統，結果通常很泛。

**好的提示結構**：

```
角色：你是一位有 10 年經驗的學校行政承辦人。
任務：請幫我把下列研習活動整理成成果報告。
約束：800 字內、含活動概述、執行情形、成效、反思四段、不用 emoji。

研習資料：
[把你整理好的要點貼進去]
```

### 3. 善用「修改」而非「重新生成」

第一次的結果通常不完美，**不要直接點「重新生成」**，而是接著說：

```
第二段太抽象，請補上具體數字（例如：「參與教師 28 人、滿意度 4.6/5」）。
```

```
標題太正式，這是給校內公告用，請改得親切一點。
```

Gemini 會在原內容上修，比每次從零生成更好。

### 4. 用「@」叫出 Google 服務

在 Gemini 輸入欄位打「**@**」可以：

- `@Drive` → 讓 Gemini 讀你 Drive 的特定檔案
- `@Gmail` → 摘要你的信件
- `@Calendar` → 看你的行程

**重要**：這些 @ 功能會把資料送進 Gemini。**請確認檔案不含敏感個資再用**。

## 5 個學校情境 × 提示詞範例

### 情境 1：把公文白話化給其他老師看

```
請把這份公文用白話文重寫，目標讀者是不熟教育部行政的新進老師。
重點：
1. 我們要做什麼？
2. 截止日是什麼時候？
3. 如果我不參加會怎樣？
4. 需要準備什麼？

請用條列式、避免「謹依」「即遵」等公文用語。

[把公文內文貼進去]
```

### 情境 2：把學生作業的常見問題整理成回饋

```
你是一位國小三年級導師。下面是 25 篇學生作業，
共同題目是「我的暑假計畫」。

請：
1. 統計學生最常見的 3 種主題類型
2. 提出 2 個寫作上的共同問題
3. 給班級整體回饋的範本（200 字）

學生作業：
[貼上學生作業文字]
```

### 情境 3：寫家長通知信

```
你幫我寫一封家長通知信，內容如下：
- 本週六校慶活動，下午 1-4 點
- 請穿便服、自備水壺
- 雨天延期，會用 LINE 群組通知
- 簽到截止前 5 分鐘到場

語氣親切、簡潔、不要超過 200 字、結尾用「敬祝 闔家平安」。
```

### 情境 4：準備教學投影片大綱

```
我要教國中三年級的「光合作用」概念，預計 50 分鐘。

請幫我規劃投影片大綱：
- 共幾張投影片
- 每張投影片標題與主要內容（1-2 句）
- 至少包含一個生活實例
- 中間有一個 5 分鐘小組討論時段
- 最後 5 分鐘有形成性評量

學生程度：對植物有基本認識、化學知識為 0。
```

### 情境 5：把會議紀錄整理成 to-do list

```
下面是上週校務會議的逐字稿，請：
1. 列出所有「誰要做什麼、什麼時候前完成」的任務
2. 依執行人分組
3. 標註「需要主任核可」的項目

逐字稿：
[貼上會議紀錄文字]
```

## 常見問題

### Q：NotebookLM 上傳的檔案會被 Google 拿去訓練嗎？

**A：對 Workspace for Education 帳號，Google 明示不會**。一般免費 Gmail 帳號的政策略有不同，學校場域請確認用 Workspace 帳號登入。

### Q：Gemini 寫的東西可以直接用嗎？

**A：不行**，永遠要人工審視。AI 可能：
- 把人名打錯
- 把日期算錯
- 把法規條文記成相近但不對的條號
- 用詞跟你校內慣例不符

**最低要求**：AI 草稿 → 人工修改 → 才能對外送出。

### Q：能不能教我寫提示詞的進階技巧？

可以的，但你應該先把「**有結構地問**」這件事練到流暢，再學 chain-of-thought、few-shot prompting 等進階技巧。

進階學習推薦：
- Google 官方 [AI Essentials](https://grow.google/ai/) 課程
- Anthropic 的 [prompt engineering 文件](https://docs.claude.com/en/docs/build-with-claude/prompt-engineering/overview)
