# 重置與移除

> ⚠️ 重置與移除都**不會**動到已建立的專案資料夾。你的歷史專案資料是安全的。

---

## 三種重置強度

### A. 只重建表單（保留歷史專案）

**情境：** 你動過表單，搞壞了；但歷史專案資料夾要保留。

**做法：**
1. Apps Script → 左側「**專案設定**」（Project Settings）
2. 往下捲到「**指令碼屬性**」（Script Properties）
3. **刪除**這四項：
   - `STARTER_FORM_ID`
   - `STARTER_RESPONSE_SHEET_ID`
   - `MILESTONE_FORM_ID`
   - `MILESTONE_RESPONSE_SHEET_ID`
4. 回到 Drive，刪掉舊的兩張表單與表單回應試算表
5. 重跑 `setupAdminWorkflow()`

---

### B. 重建表單 + 總控表（保留歷史專案）

**情境：** 總控表結構出錯，但歷史專案資料夾要保留。

**做法：** 同 A，再多刪 `CONTROL_SHEET_ID` 並把舊總控表搬到「封存」資料夾（不要直接刪，留作紀錄）。

---

### C. 完全重置（用 `resetAdminWorkflow`）

> 此函式在 P3 階段實作。本節先記錄設計與用法。

**設計：強制兩階段執行。**

```javascript
// 第一階段：dry-run，只列出「會」刪除的資源
resetAdminWorkflow(true);
```

執行後 Logger 會列出：
- 將刪除的 Apps Script 觸發器
- 將清空的 Script Properties keys
- 將刪除的表單與表單回應試算表
- 將封存（**不刪除**）的總控表
- 將**保留**的 `03_專案資料夾`（任何情況都不會自動刪）

```javascript
// 第二階段：確認 dry-run 結果後再真執行
resetAdminWorkflow(false);
```

執行後系統會：
1. 刪除兩個觸發器
2. 清空所有 Script Properties
3. 移動舊表單與回應表到 `99_系統封存_yyyyMMdd/` 子資料夾（**不直接刪除**）
4. 把舊總控表也移到封存資料夾
5. **不動** `03_專案資料夾/`

---

## 完整移除（不再使用本系統）

如果你決定完全棄用：

1. 跑 `resetAdminWorkflow(false)` 完成 C 階段重置
2. 把 Apps Script 專案改名加上 `[已停用]`（保留作紀錄）
3. **不要**刪除 Apps Script 專案（保留授權紀錄、執行歷史，未來追溯用）
4. 把 `03_專案資料夾/` 移到「校內歷史專案歸檔/」之類的位置
5. Calendar 內歷史事件可以保留（不會礙事）

---

## 手動清除清單（如果 reset 腳本壞掉）

按這個順序手動處理：

### 1. 刪除觸發器

Apps Script → 左側「觸發條件」→ 刪除 `onFormSubmit` 與 `onMilestoneFormSubmit`

### 2. 清空 Script Properties

Apps Script → 左側「專案設定」→ 指令碼屬性 → 全部刪除

### 3. 處理 Drive 資源

| 資源 | 建議處理 |
|---|---|
| `01_公文專案啟動表` 內的表單 | 移到 `99_系統封存/` |
| `01_公文專案啟動表` 內的表單回應 Sheet | 移到 `99_系統封存/` |
| `02_專案總控表` 內的總控表 | 移到 `99_系統封存/`（**不刪除**，未來歸檔查詢用） |
| `03_專案資料夾` | **完全保留** |
| `04_專案階段日期新增表` | 移到 `99_系統封存/` |
| `06_Apps Script` 內的設定備份 | 移到 `99_系統封存/` |

### 4. 處理 Calendar

- 系統建立的事件標題會有 `【活動日】`、`【成果期限】`、`【經費核銷期限】` 等字樣
- 想清理：用 Calendar 搜尋這些字串 → 批次刪除
- 想保留歷史：不動即可，過去事件不影響未來

---

## 不會被 reset 動到的東西

- ✅ 所有歷史專案資料夾 (`03_專案資料夾/`)
- ✅ 所有歷史專案的 Docs、Sheets
- ✅ Calendar 的歷史事件（除非你手動刪）
- ✅ 承辦人收過的歷史通知信

**這是設計上刻意的選擇：避免一個按鈕誤刪整年資料。**

---

## 重建後的注意事項

完全重置後重跑 `setupAdminWorkflow`：

1. ProjectCode 流水號會**重新從 001 開始**
2. 為避免新舊衝突，建議把舊總控表的「專案編號」欄改加前綴 `舊-`，例如 `舊-115-教務-001`
3. 新建立的表單網址會跟舊的**不一樣**，要重新分享給承辦人
4. 舊表單的提交不會再觸發任何事（觸發器已刪），但表單本身仍可填寫——請把舊表單在 Forms 後台關閉接受回應
