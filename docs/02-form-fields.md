# 表單欄位說明

本文件詳細說明兩張表單的所有欄位、用途、是否必填、影響哪些下游資料。

---

## 1. 公文專案啟動表

收到需要追蹤的公文或行政任務時填寫。

### 一、基本資料

| 欄位 | 必填 | 範例 | 影響的下游 |
|---|---|---|---|
| 專案年度 | ✅ | `115` | 資料夾名稱、ProjectCode 前綴 |
| 承辦處室 | ✅ | `教務處` | 資料夾名稱、ProjectCode、**多日曆派發** |
| 專案名稱 | ✅ | `AI融入教學研習` | 資料夾名稱、所有檔案名稱 |
| 承辦人 | ✅ | `王小明` | 預設任務的負責人 |
| 承辦人Email | ✅ | `wang@school.edu.tw` | 通知信收件人 |
| 協辦人員 | ⬜ | `李大華、張小芳` | 寫入專案紀錄 Docs |

### 二、公文資訊

| 欄位 | 必填 | 範例 |
|---|---|---|
| 公文主旨 | ⬜ | `辦理 AI 融入教學研習活動` |
| 來文單位 | ⬜ | `教育部資科司` |
| 公文日期 | ⬜ | 日期選擇器 |
| 公文文號 | ⬜ | `教資（一）字第 1150000001 號` |

> **若不是正式公文**（例如校內口頭交辦、上級電話通知），這四欄可空。

### 三、期限與成果要求

| 欄位 | 必填 | 範例 | 自動建立的 Calendar 事件 |
|---|---|---|---|
| 活動日期 | ⬜ | 日期選擇器 | 【活動日】當天 + 前 1 天 popup |
| 成果繳交期限 | ⬜ | 日期選擇器 | 【成果期限】當天 + 前 7 天 popup |
| 經費核銷期限 | ⬜ | 日期選擇器 | 【經費核銷期限】當天 + 前 7 天 popup |
| 是否有經費 | ⬜ | 是 / 否 / 不確定 | 影響成果檢核表「經費明細」項目的「是否需要」 |
| 核定或預估金額 | ⬜ | `50000` | 寫入專案紀錄 Docs |
| 是否需要收家長或教師回覆 | ⬜ | 是 / 否 / 不確定 | 影響成果檢核表 |
| 是否需要活動照片 | ⬜ | 是 / 否 / 不確定 | 影響成果檢核表 |
| 是否需要成果報告 | ⬜ | 是 / 否 / 不確定 | 影響成果檢核表 |
| 備註 | ⬜ | 自由文字 | 寫入總控表的「備註」欄 |

## 2. 專案階段日期新增表

既有專案後續新增日期、提醒時用。**填寫前要先從總控表複製 ProjectCode**。

| 欄位 | 必填 | 範例 | 說明 |
|---|---|---|---|
| 專案編號 | ✅ | `115-教務-001` | 用於對應總控表 |
| 專案名稱 | ✅ | `AI融入教學研習` | 系統會跟總控表比對，不符會以總控表為準 |
| 日期類型 | ✅ | 下拉選 14 種 | 見下表 |
| 日期 | ✅ | 日期選擇器 | |
| 提醒設定 | ⬜ | 多選 | 見下方說明 |
| 負責人 | ✅ | `王小明` | |
| 負責人Email | ✅ | | |
| 是否寫入待辦追蹤表 | ✅ | 是 / 否 | 預設「是」 |
| 是否建立Calendar提醒 | ✅ | 是 / 否 | 預設「是」 |
| 說明 | ⬜ | 自由文字 | |
| 備註 | ⬜ | 自由文字 | |

### 日期類型 14 種對照表

| 類型 | 對應的待辦階段 | 典型使用情境 |
|---|---|---|
| 校內協調會 | 籌備 | 跨處室協調 |
| 第一次會議 | 籌備 | 籌備會議 |
| 報名開始日 | 籌備 | 開放報名 |
| 報名截止日 | 籌備 | 收件截止 |
| 資料回收期限 | 籌備 | 家長同意書等 |
| 採購期限 | 經費 | 採購送單 |
| 經費核銷期限 | 經費 | 會計收件 |
| 活動前檢查日 | 執行 | 場地、設備、講師確認 |
| 正式活動日期 | 執行 | 活動當天 |
| 照片與成果資料回收日 | 成果 | 向講師/協辦索取資料 |
| 成果報告初稿期限 | 成果 | 內部完成草稿 |
| 成果送出期限 | 成果 | 對外送上級單位 |
| 結案檢討日 | 結案 | 內部檢討 |
| 其他 | 其他 | 不在上述清單的日期 |

> **日期類型只是分類**，不影響功能；選錯了也只是待辦表的「階段」欄位歸類有點不準。

### 提醒設定（多選）

| 選項 | Calendar 行為 |
|---|---|
| 當天提醒 | 當天 09:00 popup + email |
| 前1天提醒 | 前 1 天 09:00 popup |
| 前3天提醒 | 前 3 天 09:00 popup |
| 前7天提醒 | 前 7 天 09:00 popup |
| 前14天提醒 | 前 14 天 09:00 popup |

> **可複選**。例如重要的「成果送出期限」可以同時勾「前 14 天」+「前 7 天」+「前 3 天」+「當天」。

> **預設行為**：如果一個都沒勾，系統會自動加上「前 3 天提醒」與「當天提醒」。

## 3. 自訂表單欄位的步驟

兩張表單建立後，可以從 [Google Forms](https://docs.google.com/forms) 開啟修改。但**修改前請看下面注意事項**：

### 可以安全做的修改

- ✅ 改欄位的「題目文字」與「說明文字」
- ✅ 改選項的「顯示名稱」（不改值對應的程式邏輯）
- ✅ 加入新的「自由文字」欄位（系統會自動寫入專案紀錄 Docs）

### 會壞掉的修改

- ❌ **刪除必填欄位**（程式找不到欄位會拋例外）
- ❌ **改欄位的「題型」**（從文字改成選單、從單選改成多選）
- ❌ **改 ProjectCode 欄位名稱**（程式用名稱找欄位）

### 需要同步改程式的修改

- ⚠️ 新增**會影響下游行為**的欄位（例如「是否需要 NotebookLM 筆記本」要影響預設任務）
  - 需要在 `Code.gs` 的 `buildDefaultTasks()` 或 `createResultChecklistSheet()` 加對應邏輯

### 安全的客製方法

如果你想大幅客製，**不要直接改 Code 內建的兩張表單**，而是：

1. Fork 整個 Apps Script 專案
2. 修改 `createProjectStarterForm()` 與 `createMilestoneForm()`
3. 在 `parseFormResponse()` 加新欄位的解析
4. 在 `appendToControlSheet()`、`createProjectRecordDoc()` 加新欄位的寫入

> P3 階段我們會把表單欄位定義抽出為設定檔（YAML），讓客製不必改程式。
